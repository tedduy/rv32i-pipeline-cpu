# RTL lint

This directory owns the production RTL lint setup and generated reports.

- `verilator.vlt` contains reviewed warning policy and waivers.
- `reports/verilator_lint.log` is generated by each lint run and ignored by Git.

Verilator checks the public `rv32i_top` using the production source list in
`rtl/logical/filelist.f`. Run it from the repository root:

```bash
make lint
```

The root Make flow owns command orchestration; this directory contains only
lint-specific inputs and outputs. Keep suppressions narrow and documented.
Functional warnings must remain enabled.
